Watermill gives you three progressively higher-level APIs for working with messages. This guide starts at the lowest level — raw `Publisher` and `Subscriber` interfaces — and works up to the `Router`, which is what most production applications use. All examples here use `GoChannel`, a built-in in-memory Pub/Sub that requires no external dependencies, so you can run every snippet immediately.

## Installation

```bash
go get github.com/ThreeDotsLabs/watermill
```

<Note>
  GoChannel ships in the main module. To use Kafka, RabbitMQ, or another backend, install the matching adapter module listed in the [introduction](/introduction#supported-backends).
</Note>

## The three API layers

Watermill's APIs build on top of each other. You can use any layer independently.

<Steps>
  <Step title="Publisher and Subscriber">
    The lowest-level API. Publish messages to a topic; subscribe and receive them on a channel. Full control, minimal abstraction.
  </Step>
  <Step title="Router">
    The recommended API for most applications. Declaratively wire handlers to topics, add middleware (retry, correlation IDs, metrics), and let the Router manage goroutines and graceful shutdown.
  </Step>
  <Step title="CQRS">
    The highest-level API. Provides typed command and event buses, command and event handlers, and a clean separation between writes and reads.
  </Step>
</Steps>

## Layer 1: Publisher and Subscriber

### Creating a message

`message.NewMessage` is the only constructor you need. The UUID is optional but recommended for debugging. The payload is a plain `[]byte` — use any serialization format you like.

```go
import (
    "github.com/ThreeDotsLabs/watermill"
    "github.com/ThreeDotsLabs/watermill/message"
)

msg := message.NewMessage(watermill.NewUUID(), []byte("Hello, world!"))
```

### Publishing

`Publish` is thread-safe and accepts one or more messages per call.

```go
err := publisher.Publish("example.topic", msg)
if err != nil {
    panic(err)
}
```

### Subscribing

`Subscribe` returns a `<-chan *Message`. Each message must be acknowledged after processing. Calling `Ack()` signals successful processing; calling `Nack()` signals failure and triggers redelivery.

```go
messages, err := subscriber.Subscribe(ctx, "example.topic")
if err != nil {
    panic(err)
}

for msg := range messages {
    fmt.Printf("received message: %s, payload: %s\n", msg.UUID, string(msg.Payload))
    msg.Ack()
}
```

### Full GoChannel example

The following creates a GoChannel pub/sub, publishes messages in a goroutine, and processes them in a loop. No external services needed.

```go
package main

import (
    "context"
    "fmt"
    "time"

    "github.com/ThreeDotsLabs/watermill"
    "github.com/ThreeDotsLabs/watermill/message"
    "github.com/ThreeDotsLabs/watermill/pubsub/gochannel"
)

func main() {
    logger := watermill.NewStdLogger(false, false)

    pubSub := gochannel.NewGoChannel(gochannel.Config{}, logger)

    messages, err := pubSub.Subscribe(context.Background(), "example.topic")
    if err != nil {
        panic(err)
    }

    go func() {
        for {
            msg := message.NewMessage(watermill.NewUUID(), []byte("Hello, world!"))
            if err := pubSub.Publish("example.topic", msg); err != nil {
                panic(err)
            }
            time.Sleep(time.Second)
        }
    }()

    for msg := range messages {
        fmt.Printf("received message: %s, payload: %s\n", msg.UUID, string(msg.Payload))
        msg.Ack()
    }
}
```

## Layer 2: Router

The Router is the recommended way to build Watermill applications. It wraps publishers and subscribers, manages goroutines, handles graceful shutdown on `SIGTERM`, and provides a middleware system.

### Configure the router

Create a router and add plugins and middleware. Middleware is executed for every message that passes through the router.

```go
router, err := message.NewRouter(message.RouterConfig{}, logger)
if err != nil {
    panic(err)
}

// SignalsHandler shuts down the router gracefully on SIGTERM.
router.AddPlugin(plugin.SignalsHandler)

router.AddMiddleware(
    // CorrelationID copies the correlation ID from incoming to outgoing messages.
    middleware.CorrelationID,

    // Retry re-runs the handler on error, up to MaxRetries times.
    middleware.Retry{
        MaxRetries:      3,
        InitialInterval: time.Millisecond * 100,
        Logger:          logger,
    }.Middleware,

    // Recoverer converts panics into errors so Retry can handle them.
    middleware.Recoverer,
)
```

### Add handlers

A handler binds a subscriber topic, a subscriber, a publisher topic, and a publisher together with a handler function. Messages returned by the function are published to the output topic.

```go
// AddHandler returns a handler which can be used to add handler-level middleware
// or to stop the handler independently.
handler := router.AddHandler(
    "struct_handler",          // handler name, must be unique
    "incoming_messages_topic", // topic from which we will read events
    pubSub,
    "outgoing_messages_topic", // topic to which we will publish events
    pubSub,
    structHandler{}.Handler,
)

// Handler-level middleware is only executed for this specific handler.
handler.AddMiddleware(func(h message.HandlerFunc) message.HandlerFunc {
    return func(message *message.Message) ([]*message.Message, error) {
        log.Println("executing handler specific middleware for ", message.UUID)
        return h(message)
    }
})
```

If you only need to consume messages without producing new ones, use `AddConsumerHandler`:

```go
router.AddConsumerHandler(
    "print_incoming_messages",
    "incoming_messages_topic",
    pubSub,
    printMessages,
)
```

### Handler function types

There are two valid handler signatures:

```go
// Standalone function — use when there are no dependencies.
func printMessages(msg *message.Message) error {
    fmt.Printf(
        "\n> Received message: %s\n> %s\n> metadata: %v\n\n",
        msg.UUID, string(msg.Payload), msg.Metadata,
    )
    return nil
}

// Struct method — use when the handler needs injected dependencies.
type structHandler struct {
    // add dependencies here, e.g. a database handle
}

func (s structHandler) Handler(msg *message.Message) ([]*message.Message, error) {
    log.Println("structHandler received message", msg.UUID)
    msg = message.NewMessage(watermill.NewUUID(), []byte("message produced by structHandler"))
    return message.Messages{msg}, nil
}
```

### Run the router

`Run` blocks until the router is stopped (by context cancellation or a signal).

```go
ctx := context.Background()
if err := router.Run(ctx); err != nil {
    panic(err)
}
```

## Logging

Pass any logger implementing `watermill.LoggerAdapter`. For quick development, `NewStdLogger` works out of the box. Watermill also ships a first-class `slog` adapter:

```go
// Standard logger
logger := watermill.NewStdLogger(
    true,  // debug
    false, // trace
)

// slog adapter
logger = watermill.NewSlogLogger(slog.Default())
```
