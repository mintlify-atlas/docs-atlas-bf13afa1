The Watermill ecosystem extends well beyond the official backends maintained by Three Dots Labs. Below is a list of community-maintained libraries that add support for additional brokers, logging frameworks, observability tools, and more.

<Warning>
These libraries are not maintained by Three Dots Labs. Three Dots Labs cannot provide support or guarantee they work correctly. Do your own research before using them in production.
</Warning>

## Pub/Sub backends

The following libraries add support for messaging systems not covered by the official implementations.

<CardGroup cols={2}>
  <Card title="AMQP 1.0" href="https://github.com/kahowell/watermill-amqp10">
    AMQP 1.0 protocol support (distinct from the official AMQP 0-9-1 / RabbitMQ implementation).
  </Card>
  <Card title="Apache Pulsar" href="https://github.com/AlexCuse/watermill-pulsar">
    Apache Pulsar backend for Watermill.
  </Card>
  <Card title="Apache RocketMQ" href="https://github.com/yflau/watermill-rocketmq">
    Apache RocketMQ backend for Watermill.
  </Card>
  <Card title="CockroachDB" href="https://github.com/cockroachdb/watermill-crdb">
    CockroachDB-based Pub/Sub, maintained by the CockroachDB team.
  </Card>
  <Card title="Ensign" href="https://github.com/rotationalio/watermill-ensign">
    Ensign managed event streaming backend.
  </Card>
  <Card title="Google Cloud Pub/Sub HTTP Push" href="https://github.com/dentech-floss/watermill-googlecloud-http">
    Google Cloud Pub/Sub using HTTP push subscriptions instead of pull.
  </Card>
  <Card title="MongoDB" href="https://github.com/cunyat/watermill-mongodb">
    MongoDB-backed Pub/Sub using change streams or capped collections.
  </Card>
  <Card title="MQTT" href="https://github.com/perfect13/watermill-mqtt">
    MQTT protocol support for IoT and lightweight messaging scenarios.
  </Card>
  <Card title="NSQ" href="https://github.com/chennqqi/watermill-nsq">
    NSQ distributed messaging platform backend.
  </Card>
  <Card title="Redis Zset" href="https://github.com/stong1994/watermill-rediszset">
    Redis sorted-set-based Pub/Sub for ordered message delivery.
  </Card>
  <Card title="SQLite" href="https://github.com/davidroman0O/watermill-comfymill">
    SQLite-backed Pub/Sub for embedded or offline-first applications.
  </Card>
</CardGroup>

<Note>
If you want to build your own backend, see [Implementing a custom Pub/Sub backend](/going-further/custom-pubsub) for the interface contracts and test suite.
</Note>
