title: "Bolt embedded Pub/Sub for Watermill"
description: "Use bbolt as an embedded persistent message store inside your Go binary — no external broker, no network, no separate process required to run Watermill."
---

Bolt (bbolt) is a pure Go key/value store that embeds directly into your binary — no external server, no network, no separate process. The Watermill Bolt adapter uses it as a persistent message store, making it a good fit for small applications, CLIs, edge devices, or applications that want durable messaging without introducing infrastructure dependencies.

It is also a useful option when you already use Bolt for data storage and want to publish messages transactionally alongside other data writes.

<Note>
This backend lives in a separate module: [`github.com/ThreeDotsLabs/watermill-bolt`](https://github.com/ThreeDotsLabs/watermill-bolt). You need to install it in addition to the core `watermill` package.
</Note>

## Installation

```bash
go get github.com/ThreeDotsLabs/watermill-bolt
```

## Characteristics

| Feature               | Supported | Notes |
|-----------------------|-----------|-------|
| ConsumerGroups        | no        |       |
| ExactlyOnceDelivery   | no        |       |
| GuaranteedOrder       | no        |       |
| Persistent            | yes       |       |

## Configuration

### Shared (common) config

```go
type CommonConfig struct {
    // DB is the open bbolt database instance.
    DB *bbolt.DB

    // Bucket is the bbolt bucket name used to store messages for a topic.
    // If not set, the topic name is used as the bucket name.
    Bucket []byte
}
```

### Publisher config

```go
type PublisherConfig struct {
    CommonConfig

    // Marshaler serializes Watermill messages to byte slices for storage.
    // Defaults to JSONMarshaler.
    Marshaler Marshaler
}
```

### Subscriber config

```go
type SubscriberConfig struct {
    CommonConfig

    // Unmarshaler deserializes byte slices from Bolt into Watermill messages.
    Unmarshaler Unmarshaler

    // GenerateSubscriptionName returns the subscription name for a topic.
    // Defaults to <topic>_sub.
    GenerateSubscriptionName GenerateSubscriptionNameFn

    // PollInterval controls how often the subscriber checks for new messages.
    PollInterval time.Duration
}
```

## Setup

```go
import (
    "github.com/ThreeDotsLabs/watermill"
    "github.com/ThreeDotsLabs/watermill-bolt/pkg/bolt"
    bbolt "go.etcd.io/bbolt"
)

db, err := bbolt.Open("my-app.db", 0600, nil)
if err != nil {
    panic(err)
}
defer db.Close()

publisher, err := bolt.NewPublisher(
    bolt.PublisherConfig{
        CommonConfig: bolt.CommonConfig{DB: db},
    },
    watermill.NewStdLogger(false, false),
)
if err != nil {
    panic(err)
}
defer publisher.Close()

subscriber, err := bolt.NewSubscriber(
    bolt.SubscriberConfig{
        CommonConfig: bolt.CommonConfig{DB: db},
    },
    watermill.NewStdLogger(false, false),
)
if err != nil {
    panic(err)
}
defer subscriber.Close()
```

## Usage example

```go
package main

import (
    "context"
    "fmt"

    "github.com/ThreeDotsLabs/watermill"
    watermillbolt "github.com/ThreeDotsLabs/watermill-bolt/pkg/bolt"
    "github.com/ThreeDotsLabs/watermill/message"
    bbolt "go.etcd.io/bbolt"
)

func main() {
    db, err := bbolt.Open("app.db", 0600, nil)
    if err != nil {
        panic(err)
    }
    defer db.Close()

    logger := watermill.NewStdLogger(false, false)

    publisher, err := watermillbolt.NewPublisher(
        watermillbolt.PublisherConfig{
            CommonConfig: watermillbolt.CommonConfig{DB: db},
        },
        logger,
    )
    if err != nil {
        panic(err)
    }
    defer publisher.Close()

    subscriber, err := watermillbolt.NewSubscriber(
        watermillbolt.SubscriberConfig{
            CommonConfig: watermillbolt.CommonConfig{DB: db},
        },
        logger,
    )
    if err != nil {
        panic(err)
    }
    defer subscriber.Close()

    msg := message.NewMessage(watermill.NewUUID(), []byte(`{"event":"config_changed"}`))
    if err := publisher.Publish("config-events", msg); err != nil {
        panic(err)
    }

    messages, err := subscriber.Subscribe(context.Background(), "config-events")
    if err != nil {
        panic(err)
    }

    for msg := range messages {
        fmt.Printf("Received: %s\n", msg.Payload)
        msg.Ack()
    }
}
```

## Subscription naming

Subscriptions are created automatically when `Subscribe()` is called. The subscription name is generated by `SubscriberConfig.GenerateSubscriptionName`, defaulting to `<topic>_sub`.

Only messages published after a subscription is created are delivered to that subscription.

A topic can have multiple subscriptions, but a given subscription belongs to a single topic. If two subscribers use the same subscription name they share the message stream.

## Marshaler

Bolt stores data as raw byte slices. The default `JSONMarshaler` serializes messages as human-readable JSON — convenient for debugging. For high-volume scenarios with large messages, consider implementing a more efficient binary marshaler using the `Marshaler` / `Unmarshaler` interfaces.

## Key considerations

- **Embedded storage** — the database is a single file on disk. Bolt obtains an exclusive file lock, so only one process can open it at a time. This backend is not suitable for multi-process deployments.
- **No consumer groups** — every subscriber on a topic receives every message.
- **Polling** — the subscriber polls the database on a configurable interval. Tune `PollInterval` to balance latency against CPU usage.
- **bbolt documentation** — see [etcd-io/bbolt](https://github.com/etcd-io/bbolt) for database options such as read-only mode and custom timeout settings.
